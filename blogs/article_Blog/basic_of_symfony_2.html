1. Services in Symfony

A service is a PHP class that contains business logic. Controllers stay clean by calling services, and services can be reused across your app.

class ProductAssignService
{
    private $loanService;
    private $logger;

    public function __construct(LoanService $loanService, LoggerInterface $logger)
    {
        $this->loanService = $loanService;
        $this->logger = $logger;
    }

    public function process($params)
    {
        $this->validate();
        $this->productAssign();
        $this->log();
    }
}

‚úÖ Key Takeaway:
Logic goes in services, not controllers.
Makes your code reusable and testable.

2. Dependency Injection (DI)

Symfony automatically injects dependencies into your service:

public function __construct(LoanService $loanService, LoggerInterface $logger)

No need to manually create objects with new. Cleaner, easier to maintain and test.

3. API Interaction

Your service can call external APIs or other microservices:

$response = $this->loanService->createProductAssignment($request);
  • Sends request to LoanService
  • Receives a response
  • Keeps logic inside the service, not the controller

Why it matters:

  • Keeps code organized
  • Allows unit testing by mocking external APIs

4. Logging & Error Handling

Always log important info and handle exceptions:

try {
    $this->loanService->createProductAssignment($request);
} catch (BaseException $e) {
    $this->logger->error('Product assignment failed: ' . $e->getMessage());
}
  • Helps debug production issues
  • Keeps controllers clean
